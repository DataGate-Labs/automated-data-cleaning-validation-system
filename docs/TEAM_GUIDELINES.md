# Team Guidelines

## Purpose

This document defines the collaboration standards for the Automated Data Cleaning and Validation System project. All team members should follow these guidelines to ensure consistent development and smooth collaboration.

---

# Team Roles

## Team Lead

- Define project architecture
- Coordinate Sprint planning
- Review pull requests
- Resolve technical issues
- Track project progress

---

## Developers

- Complete assigned tasks
- Write clean and maintainable code
- Update documentation when required
- Submit Pull Requests for review

---

# Communication

Primary communication will be through:

- WhatsApp (quick discussions)
- GitHub Issues (task tracking)
- GitHub Discussions (if enabled)
- GitHub Pull Requests (code review)

---

# Git Workflow

Every new feature should be developed in its own branch.

Examples:

feature/data-profiling

feature/data-cleaning

feature/data-validation

feature/anomaly-detection

feature/report-generation

---

# Pull Request Process

Before creating a Pull Request:

- Ensure the code runs successfully.
- Add or update documentation if needed.
- Test your changes.
- Keep Pull Requests focused on a single feature or fix.

The Team Lead will review Pull Requests before merging them into the `main` branch.

---

# Coding Standards

- Follow Python PEP 8 style guidelines.
- Write meaningful variable and function names.
- Keep functions focused on a single responsibility.
- Add comments only where they improve understanding.
- Remove unused code before committing.

---

# Commit Message Guidelines

Use clear commit messages.

Examples:

- Add data profiling module
- Fix duplicate detection bug
- Improve validation logic
- Update project documentation

Avoid vague messages like:

- Update
- Changes
- Fix

---

# Repository Structure

docs/

Project documentation

src/

Application source code

tests/

Unit and integration tests

datasets/

Sample datasets for testing

configs/

Configuration files

scripts/

Utility scripts

---

# Definition of Done

A task is considered complete only if:

- Code is implemented.
- Code has been tested.
- Documentation is updated (if applicable).
- Pull Request is reviewed.
- Changes are merged into the main branch.

---

# Team Values

- Respect each other's ideas.
- Communicate openly.
- Ask questions when unsure.
- Focus on quality over speed.
- Support teammates when they are blocked.
